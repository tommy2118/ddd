Aggregates

Aggregates group Entities and Value Objects into cohesive units of consistency. They enforce domain rules, manage state changes, and act as transactional boundaries.

What are Aggregates?

An Aggregate is a cluster of domain objects that are treated as a single unit. It is defined by:

  • Aggregate Root: The main Entity that controls access to other members of the Aggregate.
  • Boundary of Consistency: All changes within an Aggregate must maintain domain rules.
  • Transaction Scope: Operations within an Aggregate are performed atomically.

Aggregates simplify complex domain logic by encapsulating related objects and ensuring consistency.

Characteristics of Good Aggregate Design

Designing Aggregates effectively requires understanding the domain rules and usage patterns. Here are some key principles:

1. Define a Clear Root

The Aggregate Root is the primary Entity responsible for managing the state and behavior of the Aggregate.

Example: In a Certification Aggregate, the Certification Entity is the root, managing linked Value Objects like dates.

2. Maintain Invariant Rules

Ensure domain rules are enforced consistently within the Aggregate boundary.

Example: A Course Aggregate ensures all lessons are completed before marking the course as completed.

3. Avoid Over-Complex Aggregates

Keep Aggregates small and focused to simplify state management and reduce the likelihood of contention in transactional systems.

Example: Separate Learner progress tracking into its own Aggregate instead of embedding it into the Course Aggregate.

Example: Aggregates in Kalkomey

At Kalkomey, Aggregates help model domain concepts like Courses, Certifications, and Learners. Here’s how they are used: 

Aggregate: Certification
- Root: Certification Entity.
- Includes: Issue Date, Expiration Date (Value Objects).
- Enforces: Expiration cannot precede Issue Date, only valid Certifications can be renewed.

Aggregate: Course
- Root: Course Entity.
- Includes: Lessons (Entities), Fee (Value Object).
- Enforces: Lessons must be completed in order, Fee must be paid before enrollment.

Aggregate: Learner
- Root: Learner Entity.
- Includes: Profile Information (Value Objects), Enrollments (Entities).
- Enforces: A learner cannot enroll in a course without prerequisites.

These Aggregates ensure domain rules are consistently applied while keeping the model manageable.

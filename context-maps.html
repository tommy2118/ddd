Context Maps

A Context Map provides a high-level view of how Bounded Contexts relate to one another. By visualizing dependencies and integration patterns, teams can better coordinate and design systems that align with business goals.

What is a Context Map?

A Context Map is a diagram that shows the relationships between different Bounded Contexts. It includes details like:

  • Boundaries: Clearly defined borders for each context.
  • Relationships: How contexts interact or depend on each other.
  • Integration Patterns: Methods used to communicate, such as REST APIs or events.

By creating a Context Map, you can identify areas of complexity, bottlenecks, and opportunities for improved collaboration.

Key Patterns in Context Maps

When defining relationships between contexts, several common patterns emerge. These patterns help clarify responsibilities and communication:

1. Customer-Supplier

One context depends on another for specific outputs. The supplier must meet the needs of the customer.

2. Shared Kernel

Two contexts share a common subset of domain logic. Teams must closely collaborate to maintain consistency.

3. Anti-Corruption Layer

A layer that translates data and interactions to prevent one context from being negatively affected by another.

Example: Kalkomey Context Map

At Kalkomey, Context Maps help us understand how our systems like LMS, E-Regulations, and HuntWise interact. Here’s a simplified Context Map: 

+-----------------------+     Customer-Supplier     +-------------------------+
|          LMS          |<------------------------->|     E-Regulations       |
| - Manages courses     |                           | - Stores regulations    |
| - Tracks learners     |                           | - Provides compliance   |
+-----------------------+                           +-------- ----------------+

+-----------------------+     Shared Kernel         +-------------------------+
|          LMS          |<------------------------->|       HuntWise          |
| - Certification data  |                           | - Hunting tools         |
| - User profiles       |                           | - Trophy tracking       |
+-----------------------+                           +-------------------------+

+-----------------------+  Anti-Corruption Layer    +-------------------------+
|       E-Regulations   |<------------------------->|     External Agency     |
| - Translates rules    |                           | - Legacy systems        |
+-----------------------+                           +-------------------------+

This map illustrates how responsibilities are distributed and highlights the integration patterns used for communication.

Value Objects

Value Objects represent concepts in your domain that do not require unique identity. They are immutable, making them simple yet powerful tools for modeling meaningful data.

What are Value Objects?

A Value Object models a concept through its attributes but lacks a unique identity. Instead, Value Objects are defined by their attributes and are considered equal if their attributes are identical. Key characteristics include:

  • Equality by Value: Two Value Objects are the same if all their attributes are equal.
  • Immutability: Once created, the state of a Value Object cannot be changed.
  • Reusability: Value Objects can be reused across Entities or Aggregates.

Value Objects help simplify your domain model by reducing unnecessary complexity and encapsulating meaningful data.

Characteristics of Good Value Object Design

Designing Value Objects effectively ensures your domain model is expressive and maintainable. Follow these principles:

1. Model Concepts, Not Identity

Use Value Objects to represent meaningful concepts like dates, coordinates, or monetary values.

Example: A Course Fee can be represented as a Value Object with amount and currency.

2. Enforce Immutability

Make Value Objects immutable to ensure their state cannot change after creation. This simplifies reasoning and prevents bugs.

Example: A Certification Date Value Object ensures integrity by remaining unchanged after issuance.

3. Add Behavior Where Needed

Encapsulate logic within Value Objects to make the model expressive. Avoid treating them as mere data holders.

Example: A GPS Coordinate Value Object can calculate distance to another coordinate.

Example: Value Objects in Kalkomey

At Kalkomey, Value Objects are used to model meaningful concepts like monetary values, coordinates, and compliance dates. Here are some examples: 

Value Object: Course Fee
- Attributes: Amount, Currency.
- Behavior: Convert currency, format for display.

Value Object: Certification Date
- Attributes: Issue Date, Expiration Date.
- Behavior: Check if expired, calculate remaining validity.

Value Object: GPS Coordinate
- Attributes: Latitude, Longitude.
- Behavior: Calculate distance to another coordinate, check proximity.

These Value Objects encapsulate data and behavior, simplifying interactions while ensuring correctness and reusability.

Interactive Exercises

Test your knowledge of Domain-Driven Design concepts with hands-on exercises. Each section provides tasks to practice Strategic Design, Tactical Modeling, and Bounded Contexts.

Strategic Design Exercises

Apply the principles of Strategic Design to real-world scenarios.

Exercise 1: Identify Core and Supporting Domains

Review Kalkomey’s systems: LMS, HuntWise, and E-Regulations. Identify which system represents the Core Domain and which are Supporting Domains. Explain your reasoning.

Exercise 2: Create a Context Map

Draw a Context Map for Kalkomey’s systems, showing the relationships between LMS, HuntWise, and E-Regulations. Include integration patterns like Customer-Supplier or Shared Kernel.

Tactical Modeling Exercises

Practice designing Entities, Value Objects, Aggregates, Domain Events, and Repositories.

Exercise 1: Design an Entity

Create an Entity for a Learner in the LMS. Include attributes like ID, Name, and Email, and behaviors like enrolling in a course.

Exercise 2: Model a Value Object

Design a Value Object for Certification Dates, including Issue Date and Expiration Date. Add behavior to check if a certification is expired.

Exercise 3: Create an Aggregate

Develop a Certification Aggregate. Include the Certification Entity as the root and related Value Objects. Ensure domain rules like “Expiration cannot precede Issue Date.”

Bounded Context Exercises

Master the art of identifying, defining, and maintaining Bounded Contexts.

Exercise 1: Identify Bounded Contexts

Examine the LMS and its integration with E-Regulations. Identify distinct Bounded Contexts and their responsibilities.

Exercise 2: Define Context Boundaries

Define the boundaries for a Hunting Regulations Context. What should it include, and how should it interact with other contexts like the LMS?

Exercise 3: Manage Context Interactions

Propose an interaction pattern (e.g., Anti-Corruption Layer or Customer-Supplier) for the LMS and HuntWise integration. Explain why you chose it.

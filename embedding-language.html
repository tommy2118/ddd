Embedding the Shared Language

A shared Ubiquitous Language is only effective if it’s consistently reflected in the codebase, systems, and workflows. Learn techniques to embed the language at every level of development.

Why Embedding the Language is Important

Embedding the shared language ensures that your systems are aligned with business goals, remain understandable, and support long-term maintainability. At Kalkomey, embedding the language helps:

Consistency

All parts of the system—from APIs to database schemas—use the same terms and concepts.

Clarity

Developers can easily understand and navigate the codebase, as it mirrors the business domain.

Scalability

A well-embedded language reduces friction when onboarding new team members or scaling systems.

Techniques for Embedding the Shared Language

Use Business Terms in Code

Ensure that the shared language is reflected in class names, methods, and variable names. For example: 

# Ruby example: Reflecting business terms in the domain model
class Certification
  attr_reader :holder, :expiration_date

  def initialize(holder, expiration_date)
    @holder = holder
    @expiration_date = expiration_date
  end

  def expired?
    Time.now > expiration_date
  end
end

Create Expressive APIs

Design APIs that reflect the Ubiquitous Language, making them intuitive for both developers and external consumers. For example: 

# Example: Certification Management API
GET /certifications/{id}  # Retrieve a certification
POST /certifications      # Issue a new certification
PUT /certifications/{id}  # Update certification details
DELETE /certifications/{id} # Revoke a certification

Align Database Schemas

Use consistent terminology in database tables and columns to mirror the domain model. For example: 

-- Example: Certification table schema
CREATE TABLE certifications (
  id SERIAL PRIMARY KEY,
  holder_id INT NOT NULL,
  issue_date DATE NOT NULL,
  expiration_date DATE NOT NULL
);

Document the Language

Maintain a central document that tracks the shared language, including its use in code, APIs, and schemas. Ensure all team members have access to this document and update it regularly.

Example: Certification Lifecycle

Here’s how a shared language is embedded across multiple layers in the Certification Management domain: 

# Domain Model
class Certification
  attr_reader :expiration_date

  def initialize(expiration_date)
    @expiration_date = expiration_date
  end

  def expired?
    Time.now > expiration_date
  end
end

# API Route
GET /certifications/{id}

# Database Schema
CREATE TABLE certifications (
  id SERIAL PRIMARY KEY,
  expiration_date DATE NOT NULL
);

Ubiquitous Language Exercises

Test your knowledge and skills in defining, refining, and embedding a shared Ubiquitous Language. These exercises are designed to reflect real-world scenarios at Kalkomey.

Exercise 1: Identifying Key Terms

Imagine you're working on the Certification Management domain. Read the scenario below and identify the key terms that should be part of the Ubiquitous Language. 

Scenario:
A user enrolls in a boating safety course provided by a training organization. Upon completion,
the organization issues a certification that includes the user's name, the course title, and the expiration date.
If the certification expires, the user must retake the course to renew their certification.

Write down the key terms and definitions you would include in the Ubiquitous Language.

Hint: Focus on nouns like "certification" and "training organization."

Exercise 2: Resolving Ambiguities

Read the conflicting definitions below for the term "user" and propose a resolution. What specific terms could you use to avoid confusion? 

Conflict:
1. "User" refers to a person enrolled in a course (student).
2. "User" refers to an administrator managing certifications.
3. "User" refers to a training organization issuing certifications.

Write your resolution. For example, you might use terms like "applicant," "administrator," or "organization."

Exercise 3: Embedding Language

Take the terms "certification" and "expiration date" from the previous exercises and embed them into:

  • A Ruby class definition.
  • A RESTful API endpoint.
  • A database schema.

Example: 

# Ruby Class
class Certification
  attr_reader :expiration_date

  def initialize(expiration_date)
    @expiration_date = expiration_date
  end

  def expired?
    Time.now > expiration_date
  end
end

# RESTful API
GET /certifications/{id}

# Database Schema
CREATE TABLE certifications (
  id SERIAL PRIMARY KEY,
  expiration_date DATE NOT NULL
);

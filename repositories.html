Repositories

Repositories abstract data access, acting as a bridge between the domain and persistence layers. They provide a clean interface for managing and retrieving domain objects.

What are Repositories?

A Repository mediates between the domain and data layers, offering an interface for accessing domain objects. It provides the following benefits:

  • Encapsulation: Hides database queries and persistence logic from the domain layer.
  • Consistency: Ensures all operations follow domain rules and invariants.
  • Testability: Simplifies unit testing by enabling in-memory or mock repositories.

Repositories make the domain model expressive and free from persistence concerns, aligning with DDD principles.

Characteristics of Good Repository Design

Designing effective Repositories requires following best practices to ensure clarity, simplicity, and alignment with the domain model. Here are key principles:

1. Follow the Aggregate Boundary

Each Repository corresponds to an Aggregate Root, ensuring that all operations respect the Aggregate’s consistency rules.

Example: A `CourseRepository` manages the `Course` Aggregate, including its associated lessons.

2. Expose Intention-Revealing Methods

Use descriptive method names that reflect the business purpose, not the technical implementation.

Example: A `findCoursesByLearner` method indicates its purpose, avoiding low-level database terminology.

3. Keep Queries Focused

Ensure queries return only the required data, avoiding unnecessary complexity or over-fetching.

Example: A `getActiveCertifications` query retrieves only active records, optimizing performance and clarity.

Example: Repositories in Kalkomey

At Kalkomey, Repositories manage domain objects like Courses, Certifications, and Learners. Here are some examples: 

Repository: CourseRepository
- Aggregate: Course.
- Methods:
  - save(Course): Persists a Course Aggregate and its associated lessons.
  - findById(CourseId): Retrieves a Course by its unique identifier.
  - findCoursesByLearner(LearnerId): Retrieves all courses enrolled by a learner.

Repository: CertificationRepository
- Aggregate: Certification.
- Methods:
  - save(Certification): Persists a Certification Aggregate.
  - getActiveCertifications(LearnerId): Retrieves active certifications for a learner.
  - findByCourseAndLearner(CourseId, LearnerId): Retrieves a specific Certification.

Repository: LearnerRepository
- Aggregate: Learner.
- Methods:
  - save(Learner): Persists a Learner Aggregate.
  - findByEmail(Email): Retrieves a Learner by their email address.

These Repositories ensure the domain model is consistently managed while abstracting persistence concerns.
